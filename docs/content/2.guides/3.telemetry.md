## Overview

Nuxt SEO collects anonymous telemetry during CI builds to help prioritize development, understand which module combinations are popular, and track ecosystem compatibility.

Nuxt SEO only sends telemetry during CI builds, never in development, testing, or local production builds.

## What is Collected

Each CI build sends a single event containing:

- **Project hash**: a SHA-256 hash of your project's root directory (first 16 characters). This allows us to count unique projects without identifying them.
- **Modules**: which Nuxt SEO modules you have installed (e.g. `@nuxtjs/sitemap`, `nuxt-og-image`)
- **Module versions**: the version of each installed module
- **Config shape**: boolean and numeric config values only (e.g. `{ sitemap: { autoLastmod: true } }`). String values are stripped.
- **Environment**: Nuxt version, [Node.js](https://nodejs.org) version, OS platform, CI provider name

No source code, file paths, URLs, API keys, or personally identifiable information is collected.

## Opting Out

Set the `NUXT_SEO_TELEMETRY_DISABLED` environment variable to disable telemetry:

```bash
NUXT_SEO_TELEMETRY_DISABLED=1
```

## Nuxt SEO Pro

Pro license holders have a separate telemetry flow tied to their license verification. Build metadata is recorded alongside the license check to power dashboard features like module status and version history. This is part of the pro license agreement and you cannot disable it independently.
